| D37 | 2026-07-01 | **Media URLs are re-homed player-side onto the origin the player reaches the server at (POL-5).** An upload's `ContentSource.src` is baked ABSOLUTE as `${MEDIA_PUBLIC_BASE}/media/<id>` (the protocol validates `src` as `z.string().url()`, so a relative path isn't an option), which defaults to `http://localhost:8080`. The player already knows a reachable server origin (the same one it dials the /player WS on, derived from `window.location`), so `resolveMediaSrc` re-homes any **loopback/unspecified-host** media URL onto it (host + port), leaving genuine external URLs untouched. | Accepted | Fixes already-uploaded media with no re-upload/migration and no dependence on `MEDIA_PUBLIC_BASE` being set correctly; loopback hosts are unreachable-by-construction from a remote box, so rewriting them is always safe (a no-op when co-located). | D38 | 2026-07-01 | **Removal = FORGET, distinct from reject/revoke and unplace (POL-14).** Two idempotent DELETEs: `DELETE /api/v1/machines/:id` forgets a machine — deletes it, all its screens, their placements + content, and its off-band credential, and CLOSES its live agent socket; `DELETE /api/v1/screens/:id` forgets one screen — dissolves any combined surface it belonged to (clearing surviving members' slices) and clears its player. The store keeps single-responsibility deletes (`deleteScreen`/`deleteContent`/`deletePlacement` per D36's prune path); `removeScreen` composes them, `deleteMachine` cascades a whole box in one op. Both routes push the cleared `server/render`s (instant path) + a fresh admin/state. A removed **machine** must re-enrol to return (gated: unknown credential → rejected, bootstrap token → fresh pending; open: re-registers brand-new); a removed **screen** reappears on its machine's next `agent/hello` if that output is still driven — so it targets STALE screens or clearing out ahead of removing the whole machine. Console surfaces "Remove" on every machine card (pending/approved/rejected) and per screen row, each behind a `window.confirm`. | Accepted | POL-14 — operators need to decommission retired boxes and clear ghost outputs. Reject/revoke only parks a machine (remembered forever) and unplace only returns a screen to the tray; neither actually forgets. |
